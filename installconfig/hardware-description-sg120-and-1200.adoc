= StorageGRID SG120 e SG1200 appliances
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Os dispositivos de serviços StorageGRID SG120 e SG1200 podem operar como um Nó de Gateway e como um Nó de Administração para fornecer serviços de balanceamento de carga de alta disponibilidade em um sistema StorageGRID. Ambos os dispositivos podem operar como Nós de Gateway e Nós de Administração (primário ou não primário) ao mesmo tempo.



== Caraterísticas do aparelho

Os aparelhos SG120 e SG1200 oferecem os seguintes recursos:

* Funções de nó de gateway ou nó de administrador para um sistema StorageGRID.
* O instalador do dispositivo StorageGRID para simplificar a implantação e a configuração de nós.
* Quando implantado, pode acessar o software StorageGRID de um nó de administrador existente ou de software baixado para uma unidade local. Para simplificar ainda mais o processo de implementação, uma versão recente do software é pré-carregada no dispositivo durante o fabrico.
* Um controlador de gerenciamento de placa base (BMC) para monitorar e diagnosticar alguns dos hardwares do dispositivo.
* A capacidade de se conetar a todas as três redes StorageGRID, incluindo a rede de Grade, a rede de Administração e a rede de Cliente:
+
** O SG120 suporta até quatro conexões de 10 ou 25 GbE à Grid Network e à Client Network.
** O SG1200 suporta até quatro conexões de 10, 25, 40 ou 100 GbE à Grid Network e Client Network (ou até quatro conexões de 200 GbE com uma 200GbE NIC opcional).

As duas unidades de estado sólido (SSDs), indicadas pelo contorno laranja, são usadas para armazenar o sistema operacional StorageGRID e são espelhadas usando RAID 1 para redundância. Quando o appliance de serviços SG120 ou SG1200 é configurado como um Admin Node, essas unidades podem ser usadas para armazenar logs de auditoria, métricas e tabelas de banco de dados. Os slots de unidade restantes estão vazios.

== Aplicações SG120 e SG1200

Você pode configurar os appliances de serviços StorageGRID das seguintes maneiras para fornecer serviços de gateway, bem como redundância de alguns serviços de administração do grid.

* Adicionar a uma grade nova ou existente como um nó de gateway
* Adicione a uma nova grade como um nó de administração primário ou não primário ou a uma grade existente como um nó de administração não primário
* Opere como um nó de gateway e um nó de administrador (primário ou não primário) ao mesmo tempo


O dispositivo facilita o uso de grupos de alta disponibilidade (HA) e balanceamento de carga inteligente para conexões de caminho de dados S3 ou Swift.

Os exemplos a seguir descrevem como você pode maximizar os recursos do dispositivo:

* Utilize dois appliances SG120 ou dois appliances SG1200 para fornecer serviços de gateway, configurando-os como Gateway Nodes.
+

IMPORTANT: A combinação de appliances de serviços com diferentes níveis de desempenho no mesmo site, como um SG110 ou SG120 com um SG1100 ou SG1200, pode causar resultados imprevisíveis e inconsistentes ao usar vários nós em um grupo de alta disponibilidade ou ao balancear a carga do cliente entre vários appliances de serviços

* Utilize dois appliances SG120 ou dois appliances SG1200 para fornecer redundância de alguns serviços de administração do grid. Faça isso configurando cada appliance como Admin Nodes.
* Utilize dois dispositivos SG120 ou dois dispositivos SG1200 para fornecer serviços de balanceamento de carga e modelagem de tráfego de alta disponibilidade, acessados por meio de um ou mais endereços IP virtuais. Faça isso configurando os dispositivos como qualquer combinação de Nós de Administração ou Nós de Gateway e adicionando ambos os nós ao mesmo grupo de alta disponibilidade.
+

IMPORTANT: Se você usar uma porta exclusiva do nó de administração e nós de gateway no mesmo grupo de alta disponibilidade, a porta exclusiva do nó de administração não será transferida em caso de falha. Consulte as instruções para  https://docs.netapp.com/us-en/storagegrid/admin/configure-high-availability-group.html["Configuração de grupos de HA"^].



Quando usados com StorageGRID storage appliances, tanto o SG120 quanto o SG1200 services appliances permitem a implantação de grids compostos exclusivamente por appliances, sem dependência de hipervisores externos ou hardware de computação.
